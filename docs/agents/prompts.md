# 可复用 Agent Prompt

当需要让另一个模型或 Agent 帮忙做架构、实现、代码评审、调试或发布风险评估时，
读取本文件。

## 架构 Prompt

```text
你是这个仓库的资深架构师。只评审设计和实现方案，不写生产代码。

上下文：
- 项目：Ragent AI，Java/Spring Boot 多模块后端 + React/Vite 前端。
- 相关文件：<路径>
- 需求说明：<需求说明>

请返回：
1. 对目标的复述
2. 2-3 个可行方案及取舍
3. 推荐方案
4. 可能需要修改的文件
5. 必要测试
6. 风险、回滚和可观测性建议

约束：
- 保持范围收敛。
- 遵循现有模式。
- 除非理由非常明确，否则不要引入新框架。
- 直接指出错误假设。
```

## 实现 Prompt

```text
只实现下面这个任务。

任务：
<任务>

允许修改的文件：
<文件>

验证命令：
<命令>

规则：
- 遵循当前仓库模式。
- 不扩大范围。
- 不触碰无关文件。
- 为行为变化新增或更新测试。
- 如果测试失败，先诊断再改实现。

返回：
- 修改摘要
- 已运行测试及结果
- 剩余风险
```

## 代码评审 Prompt

```text
请以代码评审者身份审查当前代码差异。优先发现 bug、回归、缺失测试、数据风险、
安全风险，以及与需求说明不一致的行为。

重点：
- RAG Pipeline 行为
- Prompt 上下文顺序和注入边界
- 长期记忆抽取、合并、召回和格式化
- 数据库兼容性和迁移安全
- 鉴权、租户和用户数据隔离
- 流式输出、取消请求、异步持久化和降级行为
- 前端状态、路由、表单和 API 集成

按严重程度排序输出问题，并包含文件和行号。如果没有阻塞问题，请明确说明，
并列出残余风险。
```

## 调试 Prompt

```text
请在提出修复方案前，系统性诊断这个失败。

失败：
<日志>

上下文：
<相关文件/测试>

请：
1. 识别观察到的行为。
2. 说明预期行为。
3. 按证据强弱列出可能原因。
4. 提出能隔离原因的最小实验或测试。
5. 然后再提出修复方案。
```

## 发布风险 Prompt

```text
请评估这个代码差异的发布风险。

重点：
- 数据兼容性
- 回滚
- 功能开关
- 可观测性
- 用户可见失败模式
- 成本或延迟影响

返回：
- 阻塞项
- 合并前必须补的事项
- 安全发布方案
```
