# 面试版 README 改写设计

## 目标

将根目录 `README.md` 从原项目的宣传型首页，改写为适合 `cz` 求职面试展示的项目说明。文档应让面试官快速理解项目解决的问题、系统架构、核心技术和 `cz` 实际完成的二次开发工作。

## 项目定位

- 项目名称继续使用 `Ragent AI`，避免修改代码、包名和现有图片资源。
- 维护者署名使用 `cz`，个人仓库链接使用 `https://github.com/cz1015/cz`。
- 明确说明本项目基于 Apache-2.0 协议发布的 Ragent 开源项目进行深度二次开发。
- 不将原项目整体描述为 `cz` 从零开发，不虚构生产数据、性能指标或个人职责。

## 内容结构

1. 顶部保留项目横幅，删除原仓库 Star、Fork、贡献者徽章及外链。
2. 使用简洁摘要说明企业级 Agentic RAG 的业务目标和完整链路。
3. 增加“我的工作”章节，重点展示本分支可由代码和提交记录验证的改造：
   - 结构化长期记忆抽取、持久化、召回和更新。
   - 会话摘要检查点与增量压缩策略。
   - Stable、Semi-stable、History、Ephemeral 四层 Prompt 上下文编排。
   - 长期记忆、会话摘要与流式问答 Pipeline 的整合。
   - 相关配置、数据库升级脚本和回归测试。
4. 保留并压缩原项目的核心能力、架构、模块、技术栈和界面截图。
5. 增加核心请求流程、关键设计、目录结构、快速启动和面试讲解重点。
6. 末尾保留 Apache-2.0 许可证说明和上游项目来源声明。

## 删除范围

- 删除原作者官网、在线体验、课程、社群、简历教程和其他项目推广链接。
- 删除 Star 引导、贡献者图片、Star History 和原作者个人叙事。
- 删除面向学习者的长篇行业分析、营销措辞和无法由当前仓库直接验证的宣传结论。
- 删除所有指向 `nageoffer` 个人品牌的非必要链接；仅在上游来源声明中保留原始仓库链接。

## 验收标准

- README 中维护者显示为 `cz`，个人仓库链接指向 `cz1015/cz`。
- README 不再包含原作者营销、社群或个人项目推广内容。
- README 清楚区分上游开源能力与 `cz` 的二次开发工作。
- 所有技术描述可从当前代码、配置、测试或提交差异中找到依据。
- 本地图片链接、目录锚点和示例命令保持有效。
- Markdown 结构清晰，适合面试官在数分钟内浏览。
